# Product Requirements Document (PRD)

## Project Title: Static Multimedia Website About Generative AI for Multimedia Systems

**Document Version:** 1.0  
**Language:** English  
**Prepared For:** Academic Course Project  
**Document Type:** Product Requirements Document (PRD)

---

## 1. Executive Summary

This project aims to design, build, and publish a **static multimedia-rich website** that introduces visitors to **Generative Artificial Intelligence (Generative AI)** and explains its possible applications in the field of **multimedia systems**, especially in the creation of **images, audio, and video**.

The website will function as an informative and visually engaging educational platform for a general audience, particularly students, beginners in AI, and users interested in modern digital media production. The site will present key concepts related to Generative AI, explain how generative tools work, highlight their strengths and weaknesses, and showcase real examples of their outputs through multimedia content.

The project must also include a written report, not exceeding 10 pages, that explains the website source code, the technologies used, the multimedia assets included, and clearly identifies which parts of the content were generated using AI tools. The report must also include the live URL of the deployed website hosted on a free web hosting platform.

---

## 2. Background and Problem Statement

Generative AI has become one of the most influential technologies in digital production. It is used to generate text, design images, synthesize speech, compose music, edit video, and assist in building interactive media systems. Despite its widespread use, many users still lack a clear and accessible understanding of what Generative AI is, how it works, which tools are commonly used, and what opportunities and risks it presents.

This project addresses that gap by creating an educational static website that combines clear explanations with engaging multimedia examples. The goal is not only to define Generative AI theoretically, but also to demonstrate its practical relevance in multimedia production workflows.

---

## 3. Product Vision

To create a modern, informative, multimedia-centered static website that educates users about Generative AI and demonstrates how it can be used in building multimedia systems and generating digital media assets such as images, sound, and video.

## 8. Product Type and Constraints

### 8.1 Product Type

The website must be a **static website**. This means all content is pre-built and served as files such as HTML, CSS, JavaScript, images, audio, and video without requiring a server-side application.

### 8.2 Key Constraints

1. The site must remain static.
2. Images must be in **WEBP** or **AVIF** format only for still images.
3. The final report must not exceed **10 pages**.
4. The website must be deployed on a **free hosting service**.
5. Multimedia must be optimized for web delivery while maintaining acceptable quality.
6. A portion of the content must be generated using AI tools and explicitly disclosed in the report.

## 26. Risks and Mitigation

### Risk 1: Media files become too large

**Impact:** Slow loading and poor user experience  
**Mitigation:** Compress videos, convert images to WEBP/AVIF, avoid unnecessary long clips.

### Risk 2: Overuse of AI-generated content reduces originality

**Impact:** Weak academic value or poor credibility  
**Mitigation:** Use AI selectively and refine outputs manually.

### Risk 3: Site looks visually inconsistent

**Impact:** Reduced quality perception  
**Mitigation:** Define visual identity early, use a consistent palette and typography system.

### Risk 4: Mobile layout issues

**Impact:** Poor usability on phones  
**Mitigation:** Test responsive breakpoints and optimize media containers.

### Risk 5: Report exceeds page limit

**Impact:** Does not satisfy submission rules  
**Mitigation:** Keep explanations concise and use structured sections.
